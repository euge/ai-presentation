A computer has to read this 3

  • Banks read checks, the post office sorts ZIP codes, schools scan bubble tests.
  • Every day, computers turn a messy smear of ink into the right digit.
  • By the end of these slides, you'll see the math is simpler than it looks.

Neural network

  • A stack of simple math steps that turns inputs into an answer.
  • Just dots (neurons) wired by lines (weights), in columns (layers).
  • It gets better by seeing examples.

Layer

  • A column of neurons that all work at the same time.
  • Each layer passes its results to the next one.
  • Data flows left → right.

Neuron

  • A single unit that holds one number.
  • That number is how "lit up" it is, based on what feeds in.
  • Think of it as a dimmable lightbulb — off (0) to fully bright (1).

Weight

  • A connection between two neurons with a strength dial.
  • Turn it up: that input has a louder say.
  • Turn it down (or negative): it's suppressed or inverted.
  • Training is just nudging every one of these dials.

"Neurons that fire together, wire together"

  • Paths that help the network get the right answer grow stronger.
  • Unhelpful paths fade.
  • That's the intuition behind training — you'll see it next slide.
  • (Real networks use "backpropagation" — but this model gets you 90% there.)

The image is just a grid of numbers

  • A handwritten "3" is captured as a small grid of pixels.
  • Each pixel has a brightness value between 0 (black) and 1 (white).
  • The network never sees the picture — only these numbers.
  • The job: turn 64 numbers into one answer — "it's a 3."
1.0 — fully bright (ink) 0.5 — partial brightness (edge) 0.0 — dark (background)

Each pixel becomes an input neuron

  • The 64 brightness values feed into 64 input neurons.
  • Each input neuron just holds one pixel's value.
  • Think of this layer as the eyes of the network.
  • (x₁ just means "the first input" — nothing fancy.)
input neuron xi = pixeli brightness · i = 1 to 64

Hidden neurons: multiply, add, gate

  • Each connection has a weight.
  • Multiply each input by its weight, then add the results.
  • Pass the total through a gate that keeps positives and zeroes out negatives.
  • Shown: one hidden neuron with 3 inputs (real ones have 64).
h = gate( w₁·x₁ + w₂·x₂ + w₃·x₃ ) · gate keeps positive sums, zeros out negative ones · (engineers call this "ReLU")

Output layer: same math, 10 times

  • Each output neuron does the same math: multiply, sum.
  • 10 output neurons — one per digit, 0 through 9.
  • Biggest score wins — that's the network's guess.
  • Shown: 3 of 16 hidden neurons feeding the output for "3".
output"3" = w₁·h₁ + w₂·h₂ + w₃·h₃ · the digit with the biggest score wins

Before training: the network is just guessing

  • Same network, random weights. Fresh out of the box.
  • The math still runs, but the hidden layer detects nothing meaningful.
  • It looks at a "3" and confidently blurts out "9" — it hasn't learned yet.
  • (Remember this — same network, same image, after training.)

After training: the same network, now correct

  • Same network. Same image. Only the weights changed.
  • Every weight nudged thousands of times toward being right.
  • The hidden layer now picks up curves and edges; "3" lights up strongest.
  • The circles and lines are identical — the numbers on the lines are what it learned.

Play with the knobs

  • Enough definitions — here's a real weight you can touch.
  • One neuron, 4 inputs, one dial per connection.
  • Up = amplify, down = suppress, negative = invert.
  • Multiply that by a million connections and you have a neural network.
↑ drag up to increase · ↓ drag down to decrease
output = gate( w₁·x₁ + w₂·x₂ + w₃·x₃ + w₄·x₄ )

Think lightbulbs and volume knobs

  • Every neuron is a lightbulb — brighter when its inputs are bright.
  • Every weight is a volume knob on a wire between two bulbs.
  • Turn it up for a louder say; negative to push back.

Training: tweak the knobs until it gets things right

  • Show the network a labeled example ("this is a 3").
  • Measure how wrong the guess was — that's the loss.
  • Nudge every weight a tiny bit in the direction that would reduce the loss.
  • Repeat thousands of times. (The engineering name is backpropagation.)

The secret sauce is just numbers in a file

  • When you "download a model," this is what's on disk: weights, as giant tables of decimals.
  • The architecture is a recipe — a few hundred lines of code.
  • The weights are the actual knowledge.
  • Copy the numbers and you've copied the entire trained model.
